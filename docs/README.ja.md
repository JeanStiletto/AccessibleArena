<h1>Accessible Arena</h1>

<h2>このMODについて</h2>

このMODを使うと、トレーディングカードゲーム Magic: The Gathering の最も人気があり初心者にもやさしいデジタル版である Arena をプレイできます。ゲームのほぼすべての側面に、スクリーンリーダーの完全サポートとキーボードナビゲーションを追加します。

このMODはゲームが翻訳されているすべての言語をサポートしています。さらに、ゲーム自体がサポートしていないいくつかの言語も部分的にカバーされており、これらの言語ではヘルプテキストやUIヒントなどのMOD固有の読み上げは翻訳されますが、カードやゲームデータはゲームの既定言語のままになります。

<h2>Magic: The Gathering とは</h2>

Magic は Wizards of the Coast の登録商標である、自分が魔道士となり他の魔道士たちと戦い、カードで表現された呪文を唱えるトレーディングカードゲームです。Magic には5つの色があり、それぞれ異なるゲームプレイと雰囲気のアイデンティティを表しています。Hearthstone や Yu-Gi-Oh を知っている方は多くの概念に見覚えがあるでしょう。なぜなら Magic はそれらすべてのゲームの元祖だからです。
Magic 全般についてもっと知りたい場合は、公式サイトや多くのコンテンツクリエイターが助けになってくれます。

<h2>動作要件</h2>

- Windows 10 以降
- Magic: The Gathering Arena（公式の Wizards インストーラーまたは Steam 経由でインストール）
- スクリーンリーダー（NVDA と JAWS のみテスト済み）
- MelonLoader（インストーラーが自動的に処理します）

<h2>インストール</h2>

<h3>インストーラーを使う（推奨）</h3>

1. GitHub の最新リリースから [AccessibleArenaInstaller.exe をダウンロード](https://github.com/JeanStiletto/AccessibleArena/releases/latest/download/AccessibleArenaInstaller.exe)
2. MTG Arena が起動している場合は閉じる
3. インストーラーを実行。MTGA のインストール先を検出し、必要なら MelonLoader をインストールしてMODを配置します
4. MTG Arena を起動。スクリーンリーダーから「Accessible Arena v... launched」が聞こえるはずです

<h3>手動インストール</h3>

1. [MelonLoader](https://github.com/LavaGang/MelonLoader) を MTGA フォルダーにインストール
2. 最新リリースから `AccessibleArena.dll` をダウンロード
3. DLL を MTGA の Mods フォルダーにコピー：
   - WotC 版：`C:\Program Files\Wizards of the Coast\MTGA\Mods\`
   - Steam 版：`C:\Program Files (x86)\Steam\steamapps\common\MTGA\Mods\`
4. `Tolk.dll` と `nvdaControllerClient64.dll` が MTGA のルートフォルダーにあることを確認
5. MTG Arena を起動

<h2>アンインストール</h2>

インストーラーをもう一度実行してください。MODがすでにインストールされている場合、アンインストールのオプションが提供されます。必要に応じて MelonLoader も削除できます。手動でアンインストールするには、`Mods\` フォルダーから `AccessibleArena.dll` を削除し、MTGA のルートフォルダーから `Tolk.dll` と `nvdaControllerClient64.dll` を削除してください。

<h2>Hearthstone から来た方へ</h2>

Hearthstone Access をプレイしたことがあれば、多くのものに見覚えがあるでしょう。ゲーム原理が近いだけでなく、多くの設計原則を参考にしたからです。ただし違うところもあります。

まずナビゲートするゾーンがさらに増えます。Magic には墓地、追放、その他いくつかのゾーンがあるからです。あなたの戦場はサイズに制限がなく、登場しうる大量の要素を扱いやすくするために追加の並べ替え行があります。

マナは自動で増えるのではなく、積極的にプレイする色の異なる土地カードから出てきます。これに応じてマナコストには無色部分と有色部分があり、それらを合計したものがカードを満たすために必要なコスト要件となります。

クリーチャーを直接攻撃することはできません。攻撃対象にできるのは対戦相手と、ごく一部の特別なカード（プレインズウォーカーと機体戦い）だけです。防御側としては、攻撃をブロックしてクリーチャー同士を戦わせるかを判断する必要があります。ブロックしなければダメージはあなたのプレイヤーアバターに届きますが、あなたのクリーチャーは無傷で残ることができます。さらにダメージはクリーチャーに蓄積されず、毎ターンの終了時、つまりあなたと対戦相手の両方のターン終了時に回復します。戦いに応じない対戦相手のクリーチャーに対処するには、特定のカードをプレイするか、対戦相手のライフ総量を強く圧迫して、生き残るために貴重なクリーチャーを生け贄に捧げるしか選択肢がない状況に追い込むしかありません。

このゲームには、ドロー、呪文の唱えること、戦闘など特定の行動を許すはっきりと区別された戦闘フェイズがあります。これに関連して Magic は対戦相手のターンでも何かを行うことを可能にし、むしろ奨励します。もはやただ座って何かが起こるのを待つことはありません。インタラクティブなデッキを使って敵の計画をその場で粉砕しましょう。

<h2>最初のステップ</h2>

ゲームは最初、あなたに関するいくつかのデータを入力させ、キャラクターの登録を求めます。これはゲーム内部の仕組みで動作するはずですが、うまくいかない場合はゲームのウェブサイトを代わりに使って登録できます。ウェブサイトは完全にアクセシブルです。

ゲームは Magic: The Gathering の基礎を学ぶチュートリアルから始まります。MODは標準のチュートリアルに加えて、スクリーンリーダーユーザー向けのカスタムチュートリアルヒントを追加します。チュートリアルを終えると、各色に1つずつ、合計5つのスターターデッキが報酬としてもらえます。

そこからさらにカードを入手してゲームを学ぶためにいくつかの選択肢があります：

- **カラーチャレンジ：** Magic の5色それぞれのカラーチャレンジをプレイします。各チャレンジでは4人のNPC対戦相手と戦い、最後にリアルプレイヤーとのマッチが続きます。
- **スターターデッキイベント：** 10種類の二色デッキのうち1つを選び、同じ選択肢を持つ実際のプレイヤーと対戦します。
- **Jump In：** 異なる色とテーマの20枚のカードパックを2つ選び、それらを組み合わせて1つのデッキにし、似た選択肢を持つ実際のプレイヤーと対戦します。このイベントでは無料トークンがもらえ、選んだカードは自分のものになります。
- **Spark Ladder：** ある時点で Spark Ladder が解放され、実際の対戦相手と初のランクマッチを行います。

ソーシャルメニューからメールを確認してください。多くの報酬やカードパックが含まれています。

ゲームはあなたが何をどれだけプレイしたかに応じて徐々にモードを解放していきます。進行および目標メニューでヒントやクエストが与えられ、プレイメニューでは関連するモードが強調表示されます。新規プレイヤー向けコンテンツを十分に終えると、すべての異なるモードとイベントが完全に利用可能になります。

多元宇宙のコーデックスでは、ゲームモードやメカニクスを学ぶことができます。NPE体験の進行とともに拡張されていきます。

設定のアカウントの下で、すべてのチュートリアル体験をスキップしてすべてを強制的にアンロックすれば、最初から完全な自由を得ることができます。ただし新規プレイヤーイベントをプレイすると多くのカードがもらえるので、新規プレイヤーにはおすすめです。自分が何をしているかすでに分かっている場合のみ、早めにすべてを解放してください。そうでなければ、初心者向けコンテンツは十分な楽しさと学びを提供し、上手にガイドしてくれます。

<h2>キーボードショートカット</h2>

ナビゲーションはどこでも標準的な慣習に従います：矢印キーで移動、Home/End で最初/最後にジャンプ、Enter で選択、Space で確定、Backspace で戻るまたはキャンセル。Tab/Shift+Tab もナビゲーションに使えます。Page Up/Page Down でページを変更します。

<h3>グローバル</h3>

- F1：ヘルプメニュー（現在の画面のすべてのショートカットを列挙）
- Ctrl+F1：現在の画面のショートカットを読み上げ
- F2：MOD設定
- F3：現在の画面を読み上げ
- F4：フレンドパネル（メニューから）/ デュエルチャット（デュエル中）
- F5：アップデートを確認／開始
- Ctrl+R：最後のアナウンスを繰り返し

<h3>デュエル - ゾーン</h3>

自分のゾーン：C（手札）、G（墓地）、X（追放）、S（スタック）、W（統率ゾーン）
相手のゾーン：Shift+G、Shift+X、Shift+W
戦場：B / Shift+B（クリーチャー）、A / Shift+A（土地）、R / Shift+R（非クリーチャー）
ゾーン内：左/右で移動、上/下でカード詳細を読み上げ、I で拡張情報
Shift+上/下：戦場の列を切り替え

<h3>デュエル - 情報</h3>

- T：ターン／フェイズ
- L：ライフ合計
- V：プレイヤー情報ゾーン
- D / Shift+D：ライブラリーのカード枚数
- Shift+C：相手の手札枚数
- M / Shift+M：自分／相手の土地概要
- K：フォーカスされたカードのカウンター情報
- O：ゲームログ（最近のデュエルアナウンス）
- E / Shift+E：自分／相手のタイマー

<h3>デュエル - ターゲッティングと行動</h3>

- Tab / Ctrl+Tab：ターゲットを循環（すべて／相手のみ）
- Enter：ターゲットを選択
- Space：優先権を渡す、攻撃／ブロックを確定、フェイズを進める

<h3>デュエル - フルコントロールとフェイズストップ</h3>

- P：フルコントロールの切り替え（一時的、フェイズ変更でリセット）
- Shift+P：ロックされたフルコントロールの切り替え（恒久的）
- Shift+Backspace：相手のアクションまでパスを切り替え（ソフトスキップ）
- Ctrl+Backspace：ターンスキップを切り替え（ターン全体を強制スキップ）
- 1-0：フェイズストップを切り替え（1=アップキープ、2=ドロー、3=第1メイン、4=戦闘開始、5=攻撃クリーチャー宣言、6=ブロック・クリーチャー宣言、7=戦闘ダメージ、8=戦闘終了、9=第2メイン、0=終了ステップ）

<h3>デュエル - ブラウザー（占術、諜報、マリガン）</h3>

- Tab：すべてのカードをナビゲート
- C/D：上下のゾーン間をジャンプ
- Enter：カードの配置を切り替え

<h2>トラブルシューティング</h2>

<h3>ゲーム起動後に音声が出ない</h3>

- MTG Arena を起動する前にスクリーンリーダーが動作していることを確認してください
- `Tolk.dll` と `nvdaControllerClient64.dll` が MTGA のルートフォルダーにあることを確認してください（インストーラーが自動で配置します）
- MTGA フォルダーにある MelonLoader のログ（`MelonLoader\Latest.log`）でエラーを確認してください

<h3>起動時にクラッシュする、またはMODが読み込まれない</h3>

- MelonLoader がインストールされていることを確認してください。
- ゲームが最近アップデートされた場合、MelonLoader またはMODの再インストールが必要かもしれません。インストーラーをもう一度実行してください。
- `AccessibleArena.dll` が MTGA インストール内の `Mods\` フォルダーにあることを確認してください

<h3>MODは動いていたがゲームのアップデート後に停止した</h3>

- MTG Arena のアップデートは MelonLoader のファイルを上書きすることがあります。インストーラーをもう一度実行して、MelonLoader と MOD の両方を再インストールしてください。
- ゲームの内部構造が大きく変わった場合、MODのアップデートが必要かもしれません。GitHub で新しいリリースを確認してください。

<h3>キーボードショートカットが動作しない</h3>

- ゲームウィンドウがフォーカスされていることを確認してください（クリックするか Alt+Tab で切り替え）
- F1 を押してMODが有効かどうかを確認してください。ヘルプメニューが聞こえれば、MODが動作しています。
- 一部のショートカットは特定のコンテキストでのみ動作します（デュエル用のショートカットはデュエル中のみ動作）

<h3>言語が間違っている</h3>

- F2 を押して設定メニューを開き、Enter で言語を順に切り替えてください

<h3>Windows がインストーラーまたは DLL を危険だと警告する</h3>

インストーラーと MOD DLL はコード署名されていません。コード署名証明書は年間数百ユーロかかり、無料のアクセシビリティプロジェクトには現実的ではありません。そのため Windows SmartScreen や一部のアンチウイルスは、初めてインストーラーを実行したときに警告したり、DLL を「発行元不明」としてフラグ付けしたりします。

ダウンロードしたファイルが GitHub で公開されたものと一致するか確認するために、各リリースには `AccessibleArenaInstaller.exe` と `AccessibleArena.dll` の SHA256 チェックサムが掲載されています。ダウンロードしたファイルのハッシュを計算して比較できます：

- PowerShell：`Get-FileHash <ファイル名> -Algorithm SHA256`
- コマンドプロンプト：`certutil -hashfile <ファイル名> SHA256`

ハッシュがリリースノートのものと一致すれば、ファイルは正当です。SmartScreen の警告を超えてインストーラーを実行するには、「詳細情報」を選び、「実行」をクリックしてください。

<h2>既知の問題</h2>
ゲームのほぼすべての画面をカバーしているはずですが、完全には機能しないエッジケースがある可能性があります。PayPal は非音声の違法な CAPTCHA で視覚障害者を阻むため、ゲーム内で実際のお金を使いたい場合は、晴眼者の助けを借りるか他の支払い方法を使う必要があります。
一部の特定のイベントは完全には機能しないかもしれません。実プレイヤー相手のドラフトにはまだ未サポートのロビー画面がありますが、クイックドラフトではボット相手にカードを選んでから人間の対戦相手と戦います。これは機能しており、この種の体験が好きな方にはおすすめのモードです。キューブモードは未対応です。これが何なのかすらよく分からず、ゲーム内リソースも多く消費します。ですので、時間があるときかリクエストがあれば対応する予定です。
絵文字、ペット、カードスタイル、称号などのゲームの装飾システムは現時点では部分的にしかサポートされていません。
このMODは Windows で NVDA と JAWS のみテストされており、いまだ未変更の Tolk ライブラリーに依存しています。ここで Mac や Linux の互換性をテストすることはできませんし、Prism のようなクロスプラットフォームのライブラリーはこの時点でゲームが依存する古い .NET バージョンを完全にはサポートしていません。そのため、他のプラットフォームや、未変更の Tolk では完全にサポートされていないアジア系のスクリーンリーダーのテストを手伝ってくれる人がいる場合にのみ、より幅広いライブラリーに切り替える予定です。ですので、これに取り組んでほしい方は遠慮なくご連絡ください。

現在の既知の問題リストは [KNOWN_ISSUES.md](KNOWN_ISSUES.md) を参照してください。

<h2>免責事項</h2>
<h3>その他のアクセシビリティ</h3>

このMODは主に響きが良いという理由で Accessible Arena と名乗っています。しかし現時点ではスクリーンリーダー向けのアクセシビリティMODにすぎません。視覚障害、運動障害など、より多くの障害をこのMODでカバーすることに絶対に興味があります。ただし、私自身が経験があるのはスクリーンリーダーのアクセシビリティだけです。たとえば全盲者として、色やフォントに関する問題は私にとって完全に抽象的です。ですので、そのような何かを実装してほしい場合は、ニーズを明確に説明でき、結果のテストを手伝う意思がある場合に限り、遠慮なくご連絡ください。
そうすればこのMODの名前により多くの真実を与えられることを嬉しく思います。

<h3>アプリ内課金</h3>

Arena には実際のお金にまつわる仕組みがあり、ゲーム内通貨を購入できます。これらの支払い方法は、ログインに CAPTCHA 保護を含めている PayPal を除いてほぼアクセシブルです。支払い方法の登録用にMODをアンインストールして晴眼者の助けを求めてみることもできますが、Wizards of the Coast がさらに壊れた不適切な実装をしたため CAPTCHA がアクセシビリティの悪夢となっており、これでも信頼できません。
しかし他の支払い方法は安定して動作します。私や他の人たちも物のゲーム内購入をテストしており、このシステムの使用は安全なはずです。ただしバグが起きたり、MODが誤導したりする可能性も絶対にあります。間違ったものをクリックしたり、誤った情報や不完全な情報を表示したり、Arena の内部変更により誤った動作をしたりするかもしれません。テストはできますが、あなたが実際のお金で間違ったものを買わないことを100%保証することはできません。これについて私は責任を負いませんし、公式の Arena 製品ではないのでゲーム会社も責任を負いません。この場合、払い戻しを求めようともしないでください。彼らはそれを返しません。

<h3>AIの使用</h3>

このMODのコードは Anthropic の Claude エージェントの助けを借りて100%作成されており、Opus モデルを使用しています：4.5 で始まり、開発の大半は 4.6 で行われ、リリースに向けた最終段階は 4.7 で行われました。そして最大の貢献者のおかげで少しだけ Codex も使用されました。AI の使用に関する問題は認識しています。しかし誰もがゲーム業界から望む品質や量のアクセシビリティが得られない一方で、誰もがこれらのソフトウェアをはるかに怪しいことに使っている時代において、私はそれでもツールを使うことに決めました。

<h2>クレジット</h2>

そしていまから多くの人々に感謝したいと思います。幸いなことにこれは私と AI がブラックボックスの中で行ったものではなく、周りにいる多くの人たちの大きなネットワークがあり、助け、力づけ、単に社交的で親切にしてくれました。
もし私があなたを忘れていたり、別の名前で知られたい、あるいは名前を出さないでほしい場合はDMしてください。

まず、この仕事は先駆者としての作業をしてくれた他の人々の仕事に非常に支えられており、私は Accessible Arena のためにそれを作り直すだけでした。
デザイン面では Hearthstone Access から多くを受け継ぐことができました。ゲームをプレイしたすべての人に知られているだけでなく、本当に優れた UI デザインだからです。
MOD開発の面では Zax の Modding Discord のメンバーに感謝したいです。私が単にインストールして使うだけだったすべてのツールと手順、材料を解き明かしただけでなく、AI による MOD 開発について知るべきすべてを、直接にも、公開の場での議論や他の初心者を助けることでも教えてくれました。さらに私と私のプロジェクトが存在できる場とコミュニティを与えてくれました。

大きなコード貢献については、このプロジェクトに多くの作業をしてくれた [blindndangerous](https://github.com/blindndangerous) に感謝します。プロジェクトの存続期間中、小さな厄介な問題から、より大きな UI の提案やゲーム全画面のアクセシビリティに至るまで、あらゆる種類の問題について50以上の PR をもらったと思います。
さらに、私自身のリファクタリングの上にコードの品質と保守性を確保するために実行した[大きな AI 製プロジェクト用のリファクタリングプロンプト](https://github.com/ahicks92/llm-mod-refactoring-prompts)を作成した Ahix にも感謝します。
